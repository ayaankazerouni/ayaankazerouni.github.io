Interfaces (part 2)

Rough outline:

  • interfaces help us decouple software modules (classes, in OOP) from each other
  • reminder about the Nim game
  • so far we have support for Human players—players who the game stops and asks for input on each turn
  • suppose we want support for different types of bots to play this game
  • with a slight change to use interfaces instead, we can support a variety of different “game bots”, without the game logic needing to change to support these bots
