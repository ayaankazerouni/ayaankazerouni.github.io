Mutation analysis for testing feedback

In mutation analysis, artificial bugs are injected into a working program, and the corresponding test suite is evaluated on its ability to detect those bugs.

  • Are mutants representative of the bugs that students might run into?
  • Can students use mutation-based feedback to iteratively improve their test suites?
  • Does mutation-based feedback improve students' software testing abilities?

Self-regulatory challenges faced by undergraduate computer science students

  • Help-seeking. Can students identify when they need help? What resources do they access most frequently and for what reasons? How does relate to their sense of belonging and self-efficacy in computing?
  • Personal software process management. What are the (social, cognitive, environmental) factors contributing to students' difficulties with managing large, long-running software projects? This can involve difficulties with design, development, time management, self-checking behaviour like testing and debugging, etc.

Practice and feedback with software testing

Software testing is an important self-regulatory skill involved in software development. Unfortunately, many students practice software testing as an afterthought, and face difficulties with testing that follow them into the work force.

I'm interested in the following research directions:

  • The use of mutation analysis as a device to provide students with feedback and practice with software testing [JSS '21]
  • Mining programming snapshot histories for insight into how students go about writing tests [SIGCSE '19]

Assessing and improving self-regulatory skills in software development

Novice software developers often face self-regulatory challenges when faced with large or complex programming projects (relative to what they have seen in the past). Successful completion of these projects requires declarative knowledge of how to write programs (which they typically have) as well as procedural knowledge of how to engineer software (which they typically don't have).

Changing the way this material is learned requires changing how it is taught, practiced, and assessed.

  • Explicit project milestones helped to reduce the rates of late submissions and improve project performance [SIGCSE '21]
  • Students' development habits had significant impacts on students' project performance and timeliness [ICER '17]
  • Fine-grained IDE log data yielded accurate measurements of students' development habits [ITiCSE '17]

CodeWorkout—Drill-and-practice programming exercises for novices

CodeWorkout is an online drill-and-practice system for people learning a programming language for the first time. It is free, open-source, and currently serves thousands of users at Virginia Tech and other universities.

See the project on GitHub.

  • We used CodeWorkout in our CS 1 course and explored the effects of voluntary practice of programming assignments on exam performance, controlling for individual student abilities [CompEd '19]
